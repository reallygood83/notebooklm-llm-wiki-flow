# LLM Wiki 콘텐츠 품질 기준 (창훈님 기준)

이 파일은 Claude가 `/note-wiki` 실행 결과를 평가할 때 반드시 참조하는 기준 문서입니다.

## 핵심 원칙

**의사결정에 실제로 필요한 정보만 남긴다.**
일회성 요약이 아니라, 나중에 다시 꺼내서 즉시 활용할 수 있는 지식 자산을 만든다.

## 보존 우선순위 (높은 순서)

1. **명시적 의무·금지 사항** — "해야 한다", "해서는 안 된다" 형태의 구체적 규정
2. **수치·기한·기간** — 숫자로 명시된 보존 기간, 처리 기한, 적용 범위
3. **안전·보호 규정** — 취약계층(학생·미성년자 등) 또는 고위험 상황에 대한 보호 조항
4. **고위험 의사결정 기준** — 사람이 최종 검토해야 하는 조건, 예외 처리 기준
5. **거버넌스·통제 항목** — 접근 권한, 감사 로그, 데이터 보존 통제 등 관리 체계
6. **법적 책임·배상·소유권** — 분쟁 시 책임 소재와 법적 리스크 배분

## 제거 대상 (Wiki에 저장하지 않는 것)

- 마케팅성 홍보 문구 ("업계 최고", "혁신적인" 등 일반 수식어)
- 퀴즈·용어 해설·FAQ 등 반복 학습용 콘텐츠
- 출처가 불분명한 정성적 요약
- 이미 다른 위키 페이지에 중복 저장된 내용

## 결과물 품질 체크리스트

Claude는 실행 결과를 보고하기 전에 아래를 확인한다.

- [ ] 비교 페이지(Comparison)가 생성되었는가?
- [ ] 실행 가능한 체크리스트(Checklist)가 포함되었는가?
- [ ] 원문 출처(Raw Source)가 provenance와 함께 저장되었는가?
- [ ] 생성된 콘텐츠가 위의 우선순위 기준을 실제로 반영하는가?
- [ ] 마케팅성 내용이 핵심 내용을 가리지는 않는가?

## 이 기준이 적용되는 맥락

이 체계는 GIST 행정 자동화(AX) 혁신 업무의 일환으로 운영된다.
리서치 결과물은 단순 참고 자료가 아니라, 실제 행정 판단과 의사결정에 사용되는 지식 자산이다.
따라서 모든 저장 항목은 **"이 내용을 바탕으로 당장 실행 가능한 판단을 내릴 수 있는가?"** 를 기준으로 평가한다.
